Appendix E: Field Types

CodeBase supports the following field types.

Type

Abbreviation

Information

Binary

'B' or r4bin

Binary fields are handled in the same way as Memo fields. It stores binary information. The field length is fixed at 10 bytes.

Only available with dBASE IV file compatibility.

Character

'C' or r4str

Character fields can store any type of information including binary.

Character (binary)

'Z' or r4charBin

This is the same as a Character field, but can store binary information more efficiently.

Only available with Visual FoxPro file compatibility.

Currency

'Y' or r4currency

The monetary data is stored in binary form. The field length is fixed at 8 bytes.

Only available with Visual FoxPro file compatibility.

DateTime

'T' or r4dateTime

The date and time is stored as binary data. The field length is fixed at 8 bytes.

Only available with Visual FoxPro file compatibility.

Date

'D' or r4date

Date Fields store date information only. It is stored as a string in CCYYMMDD format. e.g. March 30, 2001 is stored as “20010330”. The field length is fixed at 8 bytes.

Double

'B' or r4double

64-bit double  values are stored in binary form. The field length is fixed at 8 bytes.

Only available with Visual FoxPro file compatibility.

Floating Point

'F' or r4float

CodeBase treats this field as a Numeric field. This field is useful for compatibility with dBASE and FoxPro, which treat Floating point and Numeric fields differently. Use this field to store values that will be used in floating point calculations by dBASE or FoxPro.

General

'G' or r4gen

General fields are handled in the same way as Memo fields. It stores OLEs. The amount of information is dependent upon the size of an (unsigned int). The field length in the data file is fixed at 4 bytes in Visual FoxPro tables, otherwise it is fixed at 10 bytes.

Only available with FoxPro file compatibility.

Integer

'I' or r4int

Integer fields store 32-bit integers in binary form. The field length is fixed at 4 bytes.

Only available with Visual FoxPro file compatibility.

Logical

'L' or r4log

Logical fields store either true or false. The values that represent true are 'T', 't', 'Y', 'y'. The values that represent false are 'F', 'f', 'N', 'n'. The field length is fixed at 1 byte.

Memo

'M' or r4memo

Memo fields store the same type of information as the Character type. The amount of information is dependent upon the size of an (unsigned int). The field length in the data file is fixed at 4 bytes in Visual FoxPro tables, otherwise it is fixed at 10 bytes.

Memo (binary)

'X' or r4memoBin

This is the same as a Memo field, but can store binary information more efficiently. The field length is fixed at 4 bytes.

Only available with Visual FoxPro file compatibility.

Numeric

'N' or r4num

Numeric fields store numerical information. It is stored internally in the data file as a string of digits. This field is useful for compatibility with dBASE and FoxPro, which treat Floating point and Numeric fields differently. Use this field to store values that will NOT be used in floating point calculations by dBASE or FoxPro.

Unicode

'W' or r4unicode

Unicode fields store Unicode strings. The field is padded with nulls. When creating a table with this field type, remember to specify the length as twice the required number of Unicode characters, because each Unicode character is stored in 2 bytes.

This field type is only compatible with CodeBase. It will not be recognized by other xBASE tools. When creating a Unicode field in a FoxPro table, CODE4.compatibility must be set to 30.
