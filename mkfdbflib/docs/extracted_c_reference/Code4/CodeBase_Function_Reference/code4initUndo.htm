code4initUndo

                 Usage:    int code4initUndo( CODE4 *codebase )

          Description:    This function uninitializes CodeBase. All data, memo, and index files are flushed and closed, and any memory associated with the files is freed back to the CodeBase memory allocation pool. Furthermore, in the client-server configuration, the server connection are terminated. Once code4initUndo is called, no CodeBase function should be called unless code4init is first called.

If code4initUndo is called during a transaction, the transaction is automatically rolled back. Therefore, if a transaction is to be committed, then call code4tranCommit before code4initUndo.

               Returns:

             r4success    Success.

                       < 0    Error. See CODE4.errorCode for the exact error.

               Locking:    Locking occurs on files that require flushing. code4initUndo makes sure that any locks are unlocked when its finished.

              See Also:   code4init
