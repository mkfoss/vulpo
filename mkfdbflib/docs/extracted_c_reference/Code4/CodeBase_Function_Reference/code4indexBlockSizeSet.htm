code4indexBlockSizeSet

                 Usage:    short code4indexBlockSizeSet( CODE4 *codeBase, short blockSize )

          Description:    This function accomplishes two things. First, it sets the block size of the index files that get created. It also changes the structure of the index files so that their size can exceed traditional file size limits.

    Note

This function will only have an effect when the CodeBase library has been built with the S4FOX switch.

A larger block size can store large keys more efficiently. If your key size is greater than 100 bytes, it is recommended to use a larger block size.

The effective maximum size of the index file increases even if the block size is not changed from the default. For example, to leave the block size at 512 bytes, but increase the maximum file size, you would pass 512 as the block size parameter.

Note that if large file support has not been enabled, new index files are still created with the modified structure, but the calling application will be limited in the size of the index file it generates.

By default, the indexes created use the traditional FoxPro structure with a block size of 512.

Index files created under the new setting will not be compatible with FoxPro unless a parameter of –1 is passed to this function.

         Parameters:   

             codeBase   A pointer to a CODE4 structure.

              blockSize   The new block size when creating index files. The new value must be greater than or equal to 512, and must be a power of 2. If a blockSize of '-1' is passed in, the default FoxPro-compatible values are used.

               Returns:

             r4success    Success.

                       < 0    Error. The blockSize passed might be invalid or the index format is not FoxPro.

              See Also:   code4largeOn, Appendix D: CodeBase Limits
