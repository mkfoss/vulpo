code4collate, code4collateUnicode

                 Usage:    short code4collate( CODE4 *codeBase, short collateType )

short code4collateUnicode( CODE4 *codeBase, short collateType )

             Description:    This function controls which collating sequence to use when creating index files indexed on character fields. code4collate sets the collating sequence on 8-bit character fields. code4collateUnicode sets the collating sequence on Unicode character fields. This setting will be used when creating a new index file or adding tags to an existing index.

CodeBase supports the use of collation sequences used in Visual FoxPro. A collation sequence controls the sorting of character fields in indexing and sorting operations. The addition of collation sequences allows for the correct sorting of international languages. Not all collation sequences are available with all code pages.

    Note

This function will only have an effect when the CodeBase library has been built with the S4FOX switch defined. Compatibility with FoxPro is preserved when not using Unicode fields and using a collating sequence supported by FoxPro.

         Parameters:

             codeBase    A pointer to a CODE4 structure.

            collateType    The collation to use. Acceptable values are:

                            collate4machine   Use the collation sequence used by FoxPro 2.x file formats. This is the default value.

                              collate4general   Use the collation sequence for English, French, German, Modern Spanish, Portuguese and other Western European languages used by Visual FoxPro.

                                        collate4special   Use a custom collation sequence. A custom collation sequence may not be compatible with FoxPro. Contact the Sequiter Software consulting department for information on creating custom collation sequences.

                                                              < 0   Do nothing but return the current setting.

               Returns:    The previous setting.

              See Also:    CODE4.codePage
