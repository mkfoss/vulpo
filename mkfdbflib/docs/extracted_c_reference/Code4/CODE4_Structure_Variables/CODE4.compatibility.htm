CODE4.compatibility

                 Usage:    short CODE4.compatibility = 25

          Description:    This member specifies the version of FoxPro that data and index files are to be compatible with.

    Note

This CODE4 member will only have an effect when the CodeBase library has been built with the S4FOX switch defined.

                              Possible choices for CODE4.compatibility are as follows:

                           25   Create database files compatible with FoxPro 2.5 and before. This is the default value.

                           26   Create database files compatible with FoxPro 2.6. The format of a FoxPro 2.6-compatible index tag with a filter differs slightly from those created by all other versions of FoxPro. If an index whose tags contain filters is created with FoxPro 2.6 compatibility, it may not be read correctly by earlier or later versions of FoxPro, and vice versa.

                           30   Create database files compatible with Visual FoxPro. The use of this setting will allow the creation of data files containing the field types introduced in Visual FoxPro, like Currency, DateTime, Double, Integer, Character (binary) and Memo (binary).

 Note

This setting normally only affects how data files are created. It will be ignored when reading and updating Visual FoxPro-compatible files, but it will be used when reading earlier data files to determine if the indexes are to be read and updated in the 2.6 format, or 2.5 and earlier format.
