CODE4.memSizeSortBuffer

                 Usage:    unsigned CODE4.memSizeSortBuffer = 32768

          Description:    When sorting large amounts of information using the sort module, information often has to be temporarily stored on disk. When this condition occurs, CodeBase sort functions allocate a sequential read/write buffer to speed up this part of the sort operation. CODE4.memSizeSortBuffer specifies the size of the buffer in bytes.

Its default value is 4096 and should always be a multiple of 2048. If CODE4.memSizeSortBuffer is too large, this read/write buffer can take too much memory from the sorting and as a consequence, increase the amount of spooling that occurs.

              See Also:   CODE4.memSizeSortPool, Sort Functions
