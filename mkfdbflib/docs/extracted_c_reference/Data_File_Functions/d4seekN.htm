d4seekN

                 Usage:    int d4seekN( DATA4 *data, const char *ptr, int len)

          Description:    d4seekN searches for a record using the selected tag. If a tag has not been selected (See d4tagSelect), the first tag of the first opened index is used. Once the search value is located in the tag, the corresponding data file record is read into the record buffer. Searching always begins from the first logical record in the database as determined by the selected tag.

This function should be used when seeking for data that contains null bytes, like Unicode strings or binary data. Seeking on memo fields is not allowed.

         Parameters:

                        ptr    ptr points to a the value for which the search is conducted. See d4seek for more details.

                       len   This is the number of bytes that ptr points to. If it is greater than the tag key length, the extra bytes are ignored. If it is less than zero, it is treated as though it is equal to zero.

               Returns:    Refer to d4seek for the possible return values.

               Locking:    Refer to d4seek for locking procedures.

              See Also:   d4tagSelect, d4seek
